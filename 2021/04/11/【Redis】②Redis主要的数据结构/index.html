【Redis】②Redis主要的数据结构

发表于 2021-04-11 更新于 2021-04-24 分类于 Redis 阅读次数： Valine：

数据结构

字符串

字符串是最基础的数据结构，键都是字符串类型，而且其他几种数据结构都是在字符串类型基础上构建的

字符串类型的本质就是字符串（简单字符串、复杂字符串、数字、甚至二进制）

值最大不能超过512M

命令

（1）设置值

  • set key value
  • setex key seconds value
  • setnx key value
    • 分布式锁方法：http://redis.io/topics/distlock

（2）获取值

  • get key

（3）批量设置值

  • mset key value [key value …]

（4）批量获取值

  • mget key [key …]

（5）计数

  • incr key（自增）
    • 值不是整数，返回错误
    • 值是整数，返回自增后的结果
    • 键不存在，按照值为0自增，返回结果为1
  • decr key （自减）
  • incrby（自增指定数字）
  • decrby（自减指定数字）
  • incrbyfloat（自增浮点数）

内部编码

  • int：8字节的长整数
  • embstr：小于等于39字节的字符串
  • raw：大于39个字节的字符串

使用场景

  1. 缓存，MySQL数据的缓存
  2. 计数，统计视频播放量等
  3. 共享session，分布式系统中，共享session，集中管理
  4. 限速，验证码限制发送次数等

哈希

键值本身又是一个键值对数据结构

命令

（1）设置值

  • hset key field value

（2）获取值

  • hget key field

（3）删除field

  • hdel key field [field …]

（4）计算field个数

  • hlen key

（5）批量设置或获取filed-value

  • hmget key field [field …]
  • hmset key field value[field value …]

（6）判断field是否存在

  • hexists key field

（7）获取所有field

  • hkeys key

（8）获取所有的value

  • hvals key

（9）获取所有的field-value

  • hgetall key

hgetall 查询所有hash数据，当数据量很大时候，可能会造成redis阻塞，可以使用hscan命令，该命令会渐进式遍历哈希类型

内部编码

  • ziplist：压缩列表
    • 元素个数小于默认512个
    • 值大小小于默认64字节
    • 更加节省内存
  • hashtable：哈希表
    • 哈希类型无法满足ziplist条件
    • 读写效率下降，但是时间复杂度O(1)

使用场景

  1. 存储id:对象，对应类型，例如：userid:对象

列表

列表（list）是用来存储多个有序的字符串

最多2^32-1个元素

在列表中，可以对列表左右进行插入和弹出

列表是一种比较灵活的数据结构，可以充当栈和队列的角色

命令

  1. 添加操作

    1. 从右边插入元素

      rpush key value [value …]

    2. 从左边插入元素

      lpush key value [value …]

  2. 查找

    1. 获取指定范围元素列表

      lrange key start end

    2. 获取列表指定索引下标的元素

      lindex key index

    3. 获取列表长度

      llen key

  3. 删除

    1. 从列表左侧弹出元素

      lpop key

    2. 从列表右侧弹出元素

      rpop key

    3. 删除指定元素

      lrem key count value

    4. 按照索引范围修剪列表

      ltrim key start end

  4. 修改

    ​ lset key index newValue

  5. 阻塞操作

    ​ blpop key [key …] timeout

    ​ brpop key [key …] timeout

内部编码

  • ziplist（压缩列表）
    • 列表元素个数小于默认512个
    • 大小小于默认64字节
  • linkedlist（链表）

使用场景

  1. 消息队列
  2. 文章列表

集合

概念

和列表一样存储字符串集合，不同的是不允许重复的元素，并且是无序的，不能通过索引下标获取元素

命令

  • 集合内

    • 添加元素

      sadd key element [element …]

    • 删除元素

      srem key element [element …]

    • 计算元素个数

      scard key (时间复杂度：O(1)，不会遍历所有元素)

    • 判断元素是否存在集合中

      sismember key element

    • 随机从集合中返回指定元素

      srandmember key [count]

    • 从集合中随机弹出元素

      spop key

  • 集合间

    • 交集
      • sinter key [key …]
    • 并集
      • sunion key [key …]
    • 差集
      • sdiff key [key …]
    • 保存交集、并集、差集的结果
      • sinterstore destination key [key …]
      • sunionstore destination key [key …]
      • sdiffstore destination key [key …]

内部编码

  • intset（整数集合）
    • 当元素都是整数，且个数小于默认的512个
  • hashtable（哈希表）

使用场景

  1. 标签（tag）

有序集合

有序集合的元素不能重复，但是元素对应的value可以重复

命令

  • 集合内

    • 添加成员
      • zadd key score member [score member …]
    • 计算成员个数
      • zcard key
    • 计算某个成员的分数
      • zscore key member
    • 计算成员的排名
      • zrank key member（从低到高）
      • zrevrank key member（从高到低）
    • 删除成员
      • zrem key member [member …]
    • 增加成员的分数
      • zincr key increment member
    • 返回只当排名范围的成员
      • zrange key start end [withscores]
      • zrevrange key start end [withscores]
    • 返回指定分数范围的成员
      • zrangebyscore key min max [withscores] [limit offset count]
      • zrevrangebyscore key max min [withscores] [limit offset count]
    • 返回指定范围成员个数
      • zcount key min max
    • 删除指定排名内的升序元素
      • zremrangebyrank key start end
    • 删除指定分数范围的元素
      • zremrangebyscore key start end

  • 集合间

    • 交集
      • zinterstore destination numkeys key [key …] [weights weight [weight …]] [aggregate sum|min|max]
        • destination：交集的结果保存到这个key
        • numkeys：需要做交集计算的key数量
        • key [key …]：需要做交集计算的键
        • weights weight [weight …]：每个键的权重，分数乘以权重得到最终分数
        • aggregate sum|min|max：计算交集后，分支按照sum、min、max汇总，默认sum
    • 并集
      • zunionstore destination numkeys key [key …] [weights weight [weight …]] [aggregate sum|min|max]

内部编码

  • ziplist（压缩列表）：元素个数小于128，切大小都小于64字节，内存使用相对较少

  • skiplist（跳跃表）：读写效率会下降

使用场景

  1. 排行榜系统
